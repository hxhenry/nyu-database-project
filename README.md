# NYU Database Project

A database systems project built for coursework at New York University. This project focuses on relational database design, SQL query development, schema modeling, and database-backed application logic.

## Overview

The project demonstrates how to design and work with a structured relational database for a real-world application scenario. It covers the full database development process, from identifying entities and relationships to creating tables, defining constraints, writing SQL queries, and supporting application-level data operations.

The main goal is to show strong database fundamentals and practical backend data modeling skills that can be applied to full-stack and enterprise software development.

## Key Features

- Relational database schema design
- Entity relationship modeling
- Table creation with primary keys and foreign keys
- SQL queries for data retrieval, filtering, joins, and aggregation
- Data insertion, update, and deletion workflows
- Constraint-based data validation
- Normalized table structure for maintainability
- Query patterns for reporting and application features

## Technical Focus

- SQL
- Relational database design
- Data modeling
- Schema normalization
- Entity relationship diagrams
- Join queries
- Aggregate queries
- Transaction-oriented data workflows
- Backend-ready database structure

## Example Application Areas

This type of database design can support systems such as:

- Student and course management
- User and account management
- Inventory or order tracking
- Transaction history review
- Admin dashboards
- Search and reporting tools
- Business workflow applications

## Skills Demonstrated

- Designed structured relational data models for application workflows
- Defined relationships between entities using primary and foreign keys
- Wrote SQL queries for CRUD operations and reporting use cases
- Applied normalization principles to reduce duplication and improve data integrity
- Built database logic that can support backend services and full-stack applications
- Organized database scripts and project files for maintainability

## Why This Project Matters

Strong database design is important for building reliable software systems. This project shows the ability to think through data relationships, enforce integrity rules, write practical SQL, and structure information in a way that supports real application behavior.

## Notes

This repository represents academic and portfolio database work completed as part of NYU coursework. It is intended to demonstrate database fundamentals, SQL ability, and backend data modeling skills.
